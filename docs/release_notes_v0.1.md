# MVP v0.1 リリースノート

この版は **現場テスト開始前の凍結版** です。以降の機能追加・ルール追加は、現場テストのフィードバックを確認してから検討します。

## MVP v0.1でできること

- 保証画面コピー文から、保証日付・住所・都道府県・製品・メーカー・型番・販売店などを抽出する
- 保証開始日・保証終了日をもとに、保証開始日前・保証期間内・保証期間終了・未確認を判定する
- 保証開始日・保証終了日はカレンダー入力で扱い、内部では `YYYY/MM/DD` 文字列として保持する
- 都道府県選択から NTT東日本 / NTT西日本エリアを表示する
- 製品・メーカーを selectbox で選択し、コピー抽出結果を自動反映する
- 修理形態、保証対象外時の概算費用、修理拠点候補をCSVマスタ中心に判定する
- エアコン、パソコン、給湯器・温水器系、持込特殊製品などの主要な費用ガードを行う
- 判定診断パネルで、保証期間・概算費用・参照スクリプト・修理形態・修理拠点の状態を確認する
- `impact` により、受付不可、通話中確認、終話後確認、補足情報を分けて表示する
- 次に聞くことSTEPと終話後対応STEPを表示する
- 診断パネルの不足項目からフォーム欄へのリンクを表示する
- 対応履歴テンプレを生成する
- 実コピー文想定の fixture テストと、画面確認用 docs を備える

## まだできないこと

- 正式スクリプト本文の表示・全文検索
- 外部DBやAPIとの連携
- ユーザー認証・権限管理
- 通話録音・文字起こし
- 実受付データの永続保存
- 全メーカー・全製品の費用表完全網羅
- ブラウザ上の自動E2Eテスト

## 既知の注意点

- 正式スクリプト本文はアプリ内に保持しない
- 参照リンクは `data/master_script_links.csv` で管理できるが、本番URL登録が必要
- 実コピー文の表記ゆれは追加fixtureで継続対応する
- 費用マスタは正式表に基づき拡張中
- 修理拠点は終話後確認扱いのものがある
- 保証期間日付は誤入力防止のためカレンダー入力
- 日付が未確認の場合は空欄扱いになり、受付可否は確定しない
- CSV更新後に古い判定が残る場合は、マスタ管理タブのCSVキャッシュクリアを使う

## 現場テスト時に見るべきポイント

- 初期表示で保証開始日・保証終了日に今日の日付が勝手に入っていないこと
- 保証期間終了では、受付不可が概算費用や修理拠点より強く表示されること
- 保証開始日前では、メーカー保証または販売店・メーカー窓口への誘導が最優先になること
- エアコンのメーカー未入力では、概算費用が確定表示されないこと
- 給湯器のみでは、ガス/石油/ハイブリッドの別を確認するSTEPが出ること
- 修理拠点だけ未確定の場合、overall_status が warning にならないこと
- 次に聞くことSTEPと終話後対応STEPが混ざらないこと
- 対応履歴テンプレに保証期間判定と判定診断サマリーが入ること

## 今後の追加候補

- 本番の正式スクリプトURLを `master_script_links.csv` に登録する
- 現場で得た実コピー文を fixture に追加する
- メーカー別・製品別の概算費用ルールを正式表に沿って追加する
- 修理拠点CSVの粒度を上げる
- マスタ管理タブでCSVの差分確認や簡易バリデーションを行う
- Playwright等による画面E2E確認を追加する
- MVP v0.1 の現場フィードバックを反映して v0.2 の範囲を決める
