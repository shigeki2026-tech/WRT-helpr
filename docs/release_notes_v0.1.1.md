# v0.1.1-field-test リリースノート

## v0.1.1で追加したこと

- 通話中判定タブにオペレーター名入力を追加しました。
- 受付フォームで楽テルNO、日程調整時の連絡先、発信者区分、入電時刻を扱えるようにしました。
- 手動抽出・クリップボード直接抽出の両方で、入電時刻を `YYYY/M/D HH：MM` 形式で自動記録します。
- 回線名、修理形態、保証プランから、終話後処理タブのテンプレートを自動選択します。
- 終話後処理タブで楽テルNOを入力できます。
- テンプレート備考を修理依頼票用メモへ反映し、販売店名と電話番号の置換を支援します。
- Teams報告文を生成できるようにしました。
- 既存セッションのフォームにも新フィールドを補完するようにしました。

## v0.1からの差分

- v0.1は通話中判定と対応履歴テンプレ作成が中心でした。
- v0.1.1では、終話後処理タブを現場入力の次工程として使いやすくしました。
- オペレーター名、入電時刻、発信者区分、連絡先、楽テルNOを報告文作成に使えるようになりました。
- 業者送付コードのテンプレート選択が、手動確認前提ながら自動候補表示されるようになりました。
- 修理依頼票用メモとTeams報告文を分けて確認できるようになりました。

## まだできないこと

- Teamsへ自動送信する機能はありません。生成文を手動で貼り付けます。
- テンプレート自動選択は完全確定ではありません。意図と違う場合は手動で選択してください。
- 楽テルNOの形式チェックや重複チェックはありません。
- 日程調整時の連絡先は入力支援のみで、電話番号の厳密なバリデーションはありません。
- 正式トークスクリプト本文の表示や外部システム連携は行いません。

## 現場テストで確認すべき点

- 通話中判定タブでオペレーター名を入力できること。
- 手動抽出・クリップボード直接抽出の両方で入電時刻が自動記録されること。
- 終話後処理タブでテンプレートが想定どおり自動選択されること。
- ダブルプロテクト、物損、DPを含む保証プランでダブルプロテクト系テンプレートが優先されること。
- 楽テルNOと日程調整時の連絡先がTeams報告文へ反映されること。
- 修理依頼票用メモにテンプレート備考が反映され、販売店名と電話番号が置換されること。
- Teams報告文の貼り付け先を間違えない運用にできること。
- テンプレート自動選択が意図と違うケースを記録し、マスタ修正要否を判断すること。
