# WRT-helpr MVP v0.1 操作・導入マニュアル

このマニュアルは、修理受付支援ツール **WRT-helpr MVP v0.1 現場テスト版** を別PCへ導入し、現場テストで使うための手順書です。

対象者:
- SV / ASV
- 修理受付業務の確認担当
- 現場テスト協力者

このツールは、正式トークスクリプト本文の代替ではありません。正式な案内は必ず先方管理の最新版スクリプトを参照してください。

---

## 1. このツールでできること

WRT-helpr は、修理受付時に以下を支援します。

- 保証画面からコピーしたテキストの抽出
- 保証開始日・終了日による受付可否判定
- 製品・メーカー・都道府県のリスト入力
- 家電 / 住設判定
- 出張修理 / 持込修理 / 要確認判定
- 保証対象外時の概算費用判定
- 金額案内可否判定
- データ消去同意要否判定
- 参照すべき正式スクリプトのシート名・パート名表示
- 判定診断パネルによる不足項目表示
- 次に聞くことSTEP表示
- 終話後の対応履歴テンプレ作成

## 3. 初回セットアップ

PowerShellを開き、展開したフォルダへ移動します。

```powershell
cd C:\WRT-helpr
```

Pythonが使えるか確認します。

```powershell
py --version
```

Pythonのバージョンが表示されればOKです。

仮想環境を作成します。

```powershell
py -m venv .venv
```

PowerShellの実行ポリシーで止まる場合に備え、以下を実行します。

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

仮想環境を有効化します。

```powershell
.\.venv\Scripts\Activate.ps1
```

必要なライブラリをインストールします。

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

---

## 4. 起動方法

仮想環境が有効な状態で、以下を実行します。

```powershell
streamlit run app.py
```

ブラウザが自動で開きます。

開かない場合は、PowerShellに表示されたURLをブラウザへ貼り付けます。

通常は以下です。

```text
http://localhost:8501
```

---

## 5. 起動後の初期確認

起動したら、まず以下を確認します。

- アプリ画面が表示される
- `通話中判定` タブが表示される
- 保証開始日・保証終了日に今日の日付が勝手に入っていない
- 都道府県がリスト選択になっている
- 製品がリスト選択になっている
- メーカーがリスト選択になっている
- 判定診断パネルが表示される

保証開始日・保証終了日が空欄のままなら正常です。

## 6. 基本操作の流れ

### STEP 1: コピー情報を貼り付ける

保証管理画面などから情報をコピーし、左側のコピー情報取り込み欄へ貼り付けます。

その後、抽出・反映ボタンを押します。

抽出される主な項目:

- プラン
- 保証開始日
- 保証終了日
- お客様コード
- お客様名
- 電話番号
- 郵便番号
- 住所
- 都道府県
- WRT-NO
- 商品価格
- ジャンル
- 分類
- シリーズ
- メーカー
- 型番
- 販売店

---

### STEP 2: 受付情報フォームを確認する

中央の受付情報フォームを確認します。

特に重要な項目:

- 保証開始日
- 保証終了日
- 都道府県
- 製品
- メーカー
- 補足条件・費用判定メモ
- 販売店
- 案件区分

日付はカレンダー入力です。
コピー抽出で日付が取れた場合は、自動でカレンダーへ反映されます。

日付が分からない場合は `未確認` にしてください。

---

### STEP 3: 判定結果を見る

右側の判定結果を確認します。

確認する順番:

1. 保証期間判定
2. 判定診断
3. 次に聞くこと
4. 修理形態
5. 概算費用
6. 参照スクリプト
7. 修理拠点候補
8. 対応履歴テンプレ

保証期間判定が `受付不可` の場合は、概算費用よりも受付不可理由を優先してください。

---

## 7. 保証期間判定

保証開始日・保証終了日から、受付可否を判定します。

| 状態 | 意味 | 対応 |
|---|---|---|
| 保証開始日前 | まだWRT保証開始前 | メーカー保証または販売店・メーカー窓口へ誘導 |
| 保証期間内 | WRT保証期間内 | 通常の受付判定へ進む |
| 保証期間終了 | WRT保証終了後 | 受付不可 |
| 保証期間未確認 | 日付不足・未確認 | 保証開始日・終了日を確認 |

注意:
- 日付未入力時に今日の日付は自動入力されません。
- 未確認チェックON、またはクリア時は保証期間未確認になります。

---

## 8. 判定診断パネル

判定診断パネルは、何が不足しているか、何を次に確認すべきかを表示します。

診断対象:

- 保証期間判定
- 概算費用判定
- 参照スクリプト判定
- 修理形態判定
- 修理拠点判定

表示される内容:

- 判定エリア
- 状態
- 影響度
- 理由
- 不足項目
- 次に確認すること

不足項目には、該当フィールドへ移動するリンクが表示されます。

例:

```text
不足項目:
- メーカー欄へ移動
- 補足条件欄へ移動
```

---

## 9. 次に聞くことSTEP

通話中に確認すべき内容は、`次に聞くこと` にSTEP形式で表示されます。

例:

```text
次に聞くこと
1. メーカーを確認してください
2. 家庭用/業務用を確認してください
```

修理拠点など、終話後に確認すればよいものは `終話後対応` に分けて表示されます。

例:

```text
終話後対応
1. 終話後に担当へエスカレーションして拠点確定
```

---

## 10. 概算費用判定の注意

概算費用には以下の状態があります。

| 状態 | 意味 |
|---|---|
| confirmed | 概算費用が確定候補として表示可能 |
| pending | 追加確認が必要 |
| escalation | エスカレーション注意 |
| unavailable | 金額案内不可 |

重要な例:

- エアコンでメーカー未入力の場合、金額は表示しません。
- ダイキンエアコンは家庭用 / 業務用の確認が必要です。
- パソコンは国内メーカー / 海外メーカーで概算が変わります。
- 給湯器のみでは、ガス / 石油 / ハイブリッドの別を確認するまで金額を確定表示しません。
- ビックカメラ / ソフマップ案件は金額案内不可です。

---

## 11. 参照スクリプト

アプリは正式スクリプト本文を保持しません。

表示するのは以下だけです。

- シート名
- 該当パート
- 補足確認
- 参照リンク

参照リンクは `data/master_script_links.csv` で管理します。

URLが未登録の場合は、シート名・該当パートをもとに正式資料を手動で確認してください。

---

## 12. 対応履歴テンプレ

画面下部に対応履歴テンプレが表示されます。

内容を確認し、必要に応じてシステムへ貼り付けます。

注意:
- 保証期間終了や保証開始日前の場合は、受付不可・メーカー誘導の内容を優先してください。
- 判定診断の詳細は履歴に入れすぎず、必要最低限のサマリーにしています。

## 13. 終話後処理タブの使い方

終話後処理タブでは、通話中判定で入力した受付情報をもとに、修理依頼票用メモとTeams報告文を作成します。

操作手順:

1. `通話中判定` タブで受付情報を入力します。
2. `終話後処理` タブへ移動します。
3. テンプレートが自動選択されているか確認します。
4. 楽テルNOを入力します。
5. 日程調整時の連絡先を確認します。
6. 依頼票メモ備考を確認します。
7. Teams報告文を生成します。
8. 必要に応じてTeamsへ貼り付けます。

v0.1.1では、オペレーター名、楽テルNO、日程調整時の連絡先、発信者区分、入電時刻を扱います。入電時刻は、手動抽出またはクリップボード直接抽出を行った時点で自動記録されます。

注意事項:

- Teamsへ自動送信する機能ではありません。現時点では報告文生成までです。
- 報告文には個人情報が含まれるため、貼り付け先を間違えないでください。
- テンプレート自動選択が意図と違う場合は、必ず手動で確認してください。

## 14. マスタ管理タブ

マスタ管理タブでは、CSVマスタの内容を確認できます。

主なCSV:

| CSV | 役割 |
|---|---|
| master_product_alias.csv | 製品名の表記ゆれを正規化 |
| master_repair_type_rules.csv | 修理形態判定 |
| master_cost_rules.csv | 概算費用判定 |
| master_vendor_rules.csv | 修理拠点候補判定 |
| master_manufacturer_groups.csv | メーカーグループ |
| master_area_groups.csv | NTT東西エリア |
| master_script_links.csv | 参照スクリプトリンク |

CSVを更新した後、古い判定が残る場合は `CSVキャッシュをクリア` を押してください。

## 15. 現場テストの進め方

最初は正式運用ではなく、現場テストとして使ってください。

推奨手順:

1. `docs/demo_cases.md` の5ケースを貼り付けて確認
2. `docs/manual_checklist.md` に沿って目視確認
3. 実コピー文を1〜3件貼って確認
4. 違和感を `docs/field_test_feedback.md` に記録

記録すべき内容:

- 誤判定
- 分かりにくい表示
- 入力しづらい項目
- 足りない判定
- 追加したい項目
- 優先度

## 16. よくあるトラブル

### Pythonが認識されない

```powershell
py --version
```

`py` が使える場合は、`python` の代わりに `py` を使います。

例:

```powershell
py -m streamlit run app.py
```

### PowerShellで仮想環境を有効化できない

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

### Streamlitが起動しない

```powershell
pip install -r requirements.txt
streamlit run app.py
```

### 画面に古い判定が残る

マスタ管理タブで `CSVキャッシュをクリア` を押してください。

### 日付が勝手に入っているように見える

保証開始日・終了日の未確認チェックやクリア操作を確認してください。
未入力時に今日の日付が判定へ使われる仕様にはしていません。

## 17. 現場テストで確認するポイント

最低限、以下を確認してください。

- 初見でどこに何を入力するか分かるか
- 次に聞くことSTEPが役に立つか
- 警告が多すぎないか
- 保証期間NG時に受付不可が最優先で見えるか
- エアコン・給湯器で誤って金額案内しそうにならないか
- 履歴テンプレが実務で使える長さか
- スクリプト参照欄が分かりやすいか

---

## 18. 禁止事項・注意事項

- このアプリだけを根拠に正式案内を完結しないでください。
- 正式スクリプト本文は、必ず先方管理の最新版を確認してください。
- 概算費用はあくまで概算です。
- 未確定・要確認・エスカ表示がある場合は、表示に従って追加確認してください。
- 現場テスト中に見つかった違和感は、口頭で終わらせず記録してください。
