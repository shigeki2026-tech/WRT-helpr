# WRT-helpr — 修理受付 支援ツール

コールセンター向け修理受付をサポートする Streamlit 製ローカル支援ツールです。
AI・外部API・DBを使わず、CSVマスタとシンプルなロジックのみで動作します。

---

## 起動方法

```bash
pip install -r requirements.txt
streamlit run app.py
```

> **注意**: クリップボード読み取り機能（pyperclip）はローカルPC起動時のみ有効です。
> Streamlit Cloud など外部サーバー起動時はクリップボード取得ができないため、手動貼り付け欄を使用してください。

## MVPでできること

| 機能 | 説明 |
|------|------|
| コピー済みテキストの抽出 | 保証管理画面などのテキストをコピー→貼り付けして主要フィールドを自動抽出 |
| 都道府県リスト選択 | 47都道府県の選択式入力。選択時にNTT東日本/西日本エリアを表示 |
| 製品リスト選択 | 修理形態ルールCSVから製品候補を生成し、コピー抽出時も正規化製品を自動選択 |
| メーカーリスト選択 | メーカーグループCSVと費用CSVから候補を生成し、コピー抽出時も正規化メーカーを自動選択 |
| 保証期間ステータス判定 | 保証開始日・終了日から受付可否を判定し、修理形態・概算費用より優先表示 |
| 家電 / 住設 判定 | フォームの「家電/住設」区分をもとに修理受付フローを分岐 |
| 出張修理 / 持込修理 / 要確認 判定 | CSVマスタ（master_repair_type_rules.csv）を参照し、製品・メーカー・条件から自動判定 |
| 保証対象外時の概算費用表示 | CSVマスタ（master_cost_rules.csv）から製品・メーカー・修理形態ごとの概算を表示 |
| 金額案内可否の表示 | ビックカメラ/ソフマップ案件など、案件種別に応じて金額案内の可否を判定・表示 |
| データ消去同意要否 | パソコン・タブレット・カーナビ等は自動でデータ消去同意フローを要求 |
| 正式トークスクリプト参照先の誘導 | 対象シート名・パート名を表示（スクリプト本文は保持しない） |
| 次に確認すべき項目の一覧表示 | 修理形態・未入力項目に応じて確認質問リストを自動生成 |
| 終話後の履歴テンプレ | 受付情報をまとめた対応履歴テキストをワンクリックでコピー |
| 判定診断パネル | 5エリアの判定結果を✅/⚠️/❌でまとめ表示し、確認が必要な項目を即時提示 |

コピー情報取り込みでは、住所に含まれる都道府県を47都道府県リストから自動抽出し、受付情報フォームの都道府県選択へ反映します。都道府県が選択されると、判定結果欄に `都道府県` と `エリア: NTT東日本 / NTT西日本` が表示されます。
コピー情報取り込みでシリーズ名や分類名から製品を判定できる場合は、製品リストへ自動反映します。リスト化できない原文は「製品メモ / 原文製品名」に保持します。
コピー情報取り込みでメーカー名を抽出した場合は、メーカーリストへ自動反映し、コピー元メーカー名を「メーカー原文」として保持します。候補外メーカーは「その他・要確認」扱いになり、エアコン・パソコンでは概算費用を確定表示しません。

保証期間判定は、修理形態・概算費用より優先して表示します。保証開始日前はメーカー保証または販売店・メーカー窓口へ誘導、保証期間内は受付判定へ進む、保証期間終了後は受付不可、保証期間未確認時は保証開始日・終了日を確認します。受付不可時は、概算費用や修理拠点よりも受付不可理由を優先表示します。

保証開始日・保証終了日は `YYYY/MM/DD`、`YYYY-MM-DD`、`YYYY年M月D日` の形式に対応し、コピー抽出後はフォーム上で `YYYY/MM/DD` に正規化して保持します。

## 判定診断パネル

通話中判定タブの右カラムに **判定診断パネル** (`📊 判定診断パネル`) エクスパンダーを表示します。
確認が必要な項目がある場合は自動展開されます。

| エリア | ok 条件 | warning / error 条件 |
|--------|---------|----------------------|
| 保証期間判定 | `active` | `expired` → ❌ / `before_start` `unknown` → ⚠️（日付未入力・フォーマット不正を区別）|
| 参照スクリプト判定 | シート名が確定済み（要確認以外） | 製品・家電住設未入力・修理形態未確定 |
| 概算費用判定 | `confirmed` かつ金額確定 | `pending`（不足フィールド表示）/ `escalation`（高額エスカ注意）/ `unavailable`（金額案内不可）|
| 修理形態判定 | `出張修理` or `持込修理` | `要確認` または製品未選択・メーカー不明 |
| 修理拠点判定 | 担当エスカ以外の拠点が確定 | 担当エスカ（都道府県・製品の未入力理由を表示）|

`build_decision_diagnostics(form, result)` が呼び出され、`run_decision()` の戻り値に `diagnostics`・`overall_status` として含まれます。  
`overall_status` は `"ok"` / `"warning"` / `"error"` のいずれかです。  
`build_history_template()` の対応履歴テンプレ末尾に **【判定診断】** セクションが自動付加されます。

## CSVマスタの役割

| ファイル | 役割 |
|----------|------|
| `data/master_product_alias.csv` | 製品名のゆらぎ・略称を正規製品名に変換（例: ドライヤー・ヘアアイロン → ドライヤー） |
| `data/master_repair_type_rules.csv` | 製品・メーカー・条件から修理形態（出張/持込/要確認）を判定 |
| `data/master_cost_rules.csv` | 製品・メーカー・修理形態から保証対象外時の概算費用を判定（拡張列あり） |
| `data/master_vendor_rules.csv` | 案件種別・都道府県・製品から修理拠点候補を判定 |
| `data/master_manufacturer_groups.csv` | メーカーグループ定義（国内PC等）をCSVで管理 |
| `data/master_area_groups.csv` | NTT東日本/西日本等のエリアグループ定義 |
| `data/master_products.csv` | **旧フォールバック用**（Phase2-1以前の旧マスタ。主判定では使用しない） |

各CSVは `priority` 昇順で評価し、最初にヒットしたルールを採用します（first-match wins）。
`enabled=0` の行はスキップされます。
CSV更新後に画面へ古い判定が残る場合は、マスタ管理タブの **CSVキャッシュをクリア** を押してください。CSVローダーはファイル更新時刻も参照するため、通常は更新後に自動で再読み込みされます。

### master_cost_rules.csv 拡張列

| 列名 | 意味 |
|------|------|
| `required_fields` | 費用を確定表示する前に必須のフォームフィールド（例: `manufacturer;extra_condition`）。未入力時は `cost_status=pending` を返す |
| `cost_status` | `confirmed` / `pending` / `escalation` — 費用確定状態 |
| `guidance_scope` | `always`（常時表示）/ `eu_asked_only`（EUから質問があった場合のみ案内）|
| `required_questions` | pendingのときに次に確認すべき質問文（UIに表示） |
| `customer_notice` | お客様へ案内してよい補助文 |
| `internal_note` | オペレーター内部向け注意（画面下部に小さく表示） |

### 概算費用 4状態の表示仕様

| 状態 | 表示色 | 発生条件 |
|------|--------|----------|
| `confirmed` | 緑 | 費用確定・案内可 |
| `pending` | 黄 | required_fields が未入力、または修理形態が要確認 |
| `escalation` | オレンジ | 高額等でエスカレーション推奨（needs_escalation=1） |
| `unavailable` | 赤 | ビックカメラ/ソフマップ等で金額案内不可 |

### エアコンの費用制御

- **メーカー未入力** → pending「メーカーを確認してください」
- **メーカー未入力時は概算費用を表示しません**（汎用出張費用へのフォールバックもブロック）
- **ダイキン + 家庭用/業務用未入力** → pending「家庭用・業務用を確認してください」
- **ダイキン + 家庭用** → 7,000円～16,000円前後
- **ダイキン + 業務用** → 15,000円～22,000円前後
- **ガス漏れ検査**（ダイキン + extra_condition="ガス漏れ"）→ 30,000円前後（eu_asked_only: EUから質問があった場合のみ案内）

### パソコンの費用制御

- **メーカー未入力** → pending「国内・海外メーカーをご確認ください」
- **国内メーカー**（パナソニック・富士通・NEC等）→ 2,000円～9,000円
- **海外メーカー**（Dell・HP・Lenovo等）→ 12,000円前後

### 給湯器・温水器系の費用制御

- **エコキュート + ダイキン** → 15,000円～20,000円前後
- **エコキュート + その他国内メーカー** → 8,000円～10,000円前後
- **ガス給湯器 / 石油給湯器** → 5,000円～7,000円前後
- **ハイブリッド給湯器** → 8,000円～10,000円前後
- **エネファーム** → 5,000円～7,000円前後。ガス会社聴取必須。ガス会社を通して各メーカーへ手配します。
- **電気温水器 / 電気暖房温水ボイラー** → 8,000円～10,000円前後
- **給湯器のみ** → pending。ガス給湯器・石油給湯器・ハイブリッド給湯器の別を確認するまで概算確定表示しません。

給湯器・温水器系はいずれも、設置状況・診断内容により実際の金額が異なります。

### 持込特殊製品の費用制御

- **デジカメ / 一眼レフカメラ / ビデオカメラ** → 2,000円前後
- **電子ピアノ脚なし / ピアノ脚なし + ローランド** → 6,000円～15,000円前後
- **電子ピアノ脚なし / ピアノ脚なし + ローランド以外** → 2,000円～5,000円前後
- **Airdog** → 7,000円～10,000円前後。検証・返送料含む概算で、サイズにより送料が高くなる見積です。
- **パワーウエーブ系（FITプロジェクト / TKクリエイト）** → 4,000円～5,000円前後。0800-919-0757 の平日持込製品のため対応注意。リモコン等の付属品は対象外です。
- **パイオニアAV製品** → 16,000円前後。概算費用に不承の場合はエスカレーションしてください。

パイオニアAV製品ルールは `AV製品` のみ対象です。カーナビ・カーオーディオはこのルールから除外し、別ルールまたは汎用持込として扱います。

## 自動テスト実行

```bash
# 依存パッケージのインストール（pytest を含む）
pip install -r requirements.txt

# ロジック単体テスト
python -m pytest tests/test_decision_rules.py -v

# 実コピー文想定テスト
python -m pytest tests/test_fixture_extraction.py -v

# 全テスト
python -m pytest -v

# スタンドアロンで実行（pytest 不要）
python tests/test_decision_rules.py
```

ロジック単体テストは `tests/test_decision_rules.py`、保証画面コピー文を想定した統合テストは `tests/test_fixture_extraction.py` に分けています。
fixtureテストは、実コピー文のラベル名・区切り文字の表記ゆれ検証用です。新しい実コピー文を入手したら `tests/fixtures/` に追加し、ラベル揺れが見つかった場合は `extract_fields_from_pasted_text()` のラベル候補へ追加してください。
`check()` 内で `assert` しているため、期待値と実際値が異なる場合は pytest が確実に FAIL を報告します。

## 手動テスト項目

動作確認時は以下のシナリオを入力して結果を確認してください。

| 入力 | 期待結果 |
|------|----------|
| シリーズ: `ドライヤー・ヘアアイロン` | 製品正規化 → **ドライヤー** |
| 製品: `ドライヤー` | 修理形態 → **持込修理** |
| 製品: `ドライヤー`（汎用） | 概算費用 → **2,000円～5,000円前後** |
| 製品: `洗濯機` | 修理形態 → **出張修理** |
| 製品: `エアコン` / メーカー: `ダイキン`（家庭用） | 概算費用 → **7,000円～16,000円前後** |
| 都道府県: `滋賀県` / 製品: `洗濯機` | エリア → **NTT西日本** / 修理拠点 → **ユナイトサービス㈱** |
| 住所コピー: `東京都新宿区...` | 都道府県選択 → **東京都** / エリア → **NTT東日本** |
| 案件区分: `ビックカメラ案件` または `ソフマップ案件` | 金額案内 → **不可**（ソフマップ修理センター） |
| 製品: `パソコン` | データ消去同意 → **必要** |

## 注意事項

- **正式トークスクリプトの本文はアプリ内に保持しません。**
  シート名・パート名の参照先誘導のみを行います。
- 必ず先方管理の最新版 Excel（正式トークスクリプト）を参照して案内してください。
- このアプリは正式スクリプト本文の代替ではなく、**参照先誘導ツール**です。
- プリンターの扱いは、MVPプロンプトでは持込修理としています。
  ただし実業務では型番・メーカーにより要確認の可能性があるため、**正式運用前に先方確認が必要**です。
