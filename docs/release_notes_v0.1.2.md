# v0.1.2-field-test リリースノート

## v0.1.2で追加したこと

- 終話後処理タブに `Teamsチャットへ送信` ボタンを追加しました。
- 生成したTeams報告文を、Microsoft Graph PowerShell 経由で既存Teamsチャットへ投稿できるようにしました。
- 送信先は `config/teams_config.json` または環境変数 `WRT_TEAMS_CHAT_ID` で管理します。
- 実際の `chat_id` をGitに含めないため、`config/teams_config.example.json` をサンプルとして追加しました。
- 送信結果ログをセッション内に保持し、画面上で直近3件を確認できます。

## 誤送信防止

- 送信前に `送信内容と送信先を確認しました` のチェックが必須です。
- Teams送信が無効、または `chat_id` が未設定の場合、送信ボタンは押せません。
- 送信先チャット名を画面に表示し、送信前に確認できるようにしました。

## まだできないこと

- 自動送信はしません。
- 一括送信はしません。
- 送信ログのローカルファイル保存はしません。ログはStreamlitセッション内のみです。
- Teamsチャットの検索や新規作成はしません。送信先は既存チャットの `chat_id` を事前設定します。

## 既知の注意点

- 初回送信時は Microsoft Graph PowerShell のログインが必要になる場合があります。
- Microsoft Graph の delegated permission `ChatMessage.Send` が必要です。
- Teams送信に失敗した場合は、従来どおり報告文をコピーして手動投稿してください。
- 報告文には個人情報が含まれる可能性があるため、送信先チャット名と本文を必ず確認してください。
